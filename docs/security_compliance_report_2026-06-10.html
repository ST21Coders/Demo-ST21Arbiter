What this harness tested

The harness runs nine layers, each a focused pytest or Playwright suite: e2e, fuzz, auth, headers, llm, dos, logic, logging_audit, fault. Each layer's results land in a shared coverage matrix aggregator (src/coverage/builder.py) that emits the daily PDF report and the per-run findings JSON. This document is the compliance-side view of that same data: which categories of risk the nine layers exercise, and which they deliberately do not.

Methodology & reproduction

Harness shape

The harness is a black-box test suite that runs against the deployed dev environment. It does not have access to backend source at runtime — every assertion is made on what the deployed API, the deployed SPA, or the deployed AgentCore runtimes actually return. A nightly cost cap (default $1.00) gates LLM-spending probes so a runaway fan-out cannot exhaust the demo budget. Layers that touch the LLM surface (llm, fault) account against that cap; the other seven layers are zero-cost.

The nine layers

LayerWhat it tests
e2ePlaywright sweeps the SPA per persona and asserts page-level access.

authJWT forgery, IDOR, brute-force, session fixation, password reset.

logicWorkflow bypass, race conditions, sensitive-field exposure.
logging_auditAudit-log writes + CloudWatch redaction + log injection.
faultFail-closed, error propagation, partial failure, LLM output safety.

Run cadence

The full harness runs on a daily cadence against the dev environment. Each run produces a report.html + summary.md + report.json in tests-adversarial/test-reports/<run-id>/, and the dated PDF in docs/adversarial_run_findings_<date>.pdf. After every "build block" lands (one new test layer or a refresh of an existing one), this compliance matrix is regenerated so the coverage percentage moves with reality.

npm run test:all — run the nine layers and aggregate.
